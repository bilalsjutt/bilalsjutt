# Bilal
 
I'm an AI engineer, mostly working on agent systems these days. I run a startup called KortexMail, an AI email tool, where the main thing I built is the memory layer — the part that lets the assistant actually remember context instead of starting over every session. It runs on a mix of a vector store and a graph database working together, plus an agent harness on top that handles the reasoning. It's live at cortexmail.com if you want to see it in action.
 
Building something that's supposed to get smarter over time got me thinking a lot about how it could be attacked instead — bad data slipping into memory, agents misusing the tools they have access to. That's turned into a real interest of mine, and it's where I want to take things next: red teaming and security work for AI systems that are being put into production.
 
What I bring to that:
 
- Designing memory systems for AI agents, combining vector stores and knowledge graphs so an agent retains context instead of losing it between sessions
- Building multi-tool agent harnesses (LangGraph, DSPy) that can reason across dozens of tools without falling apart
- Shipping the backend and infra side myself — FastAPI, AWS, Docker, reverse proxies, deployment, all of it
- AI red teaming — finding memory poisoning, tool misuse, and other attack paths in agentic systems before they become a production problem
- Freelance work building RAG pipelines and custom agents for clients on Upwork and Fiverr
I can design and build agent systems for enterprise use end-to-end, and I do AI red teaming on those same systems to find where they'd break in production.
